Python/Pandas 기초 보강

시험 코드는 길고 멋진 코드보다, 짧고 정확한 패턴을 안정적으로 떠올리는 힘이 중요합니다. 이 페이지는 "왜 이 문법을 쓰는지", "언제 쓰는지", "실수하면 어디서 틀리는지"를 중심으로 정리했습니다.

1. 시험 코딩의 기본 흐름

대부분의 작업형 1, 3 문제는 데이터를 불러온 뒤 조건을 걸고, 집계하고, 마지막 숫자 하나를 출력하는 구조입니다. 그래서 처음부터 모델을 외우기보다 read_csv -> 확인 -> 필터링 -> 계산 -> print 흐름을 몸에 붙이는 것이 먼저입니다.
import pandas as pd

df = pd.read_csv('data/titanic.csv')
print(df.shape)
print(df.head())
print(df.info())
  • df는 관례적으로 DataFrame을 담는 변수명입니다. 이름 자체보다 "표 전체"라는 감각이 중요합니다.
  • print()를 마지막에 빼먹으면 자동 채점기가 볼 출력이 없습니다.
  • 웹 실행기에서는 데이터 경로를 data/titanic.csv처럼 씁니다.

2. 시험에 가장 자주 나오는 Pandas 패턴

열 선택
df['Fare']
한 열만 꺼내 평균, 중앙값, 결측치 개수를 계산할 때 씁니다.
조건 필터링
df[df['Sex'] == 'female']
특정 집단만 남기고 계산할 때 씁니다.
복합 조건
df[(df['Pclass'] == 1) & (df['Age'] >= 30)]
조건 두 개 이상은 괄호와 &, |를 사용합니다.
파생 변수
df['Family'] = df['SibSp'] + df['Parch']
기존 열을 조합해 새 열을 만들 때 씁니다.
결측치 확인
df.isna().sum()
빈칸이 어디에 많은지 확인합니다.
그룹 집계
df.groupby('Pclass')['Fare'].mean()
등급별 평균처럼 집단별 통계를 계산할 때 씁니다.

3. 결측치와 정렬은 왜 조심해야 하나

결측치를 삭제할지 채울지는 정답값을 바꿉니다. 시험 문제에서 "평균으로 대체", "제거", "최빈값으로 대체"라고 말하면 그대로 따라야 합니다. 작업형 2에서는 train 기준으로 만든 대체값을 test에 적용해야 데이터 누수를 피할 수 있습니다.
age_mean = df['Age'].mean()
df['Age'] = df['Age'].fillna(age_mean)

fare_top = df['Fare'].quantile(0.9)
answer = df[df['Fare'] >= fare_top]['Age'].mean()
print(round(answer, 2))
  • fillna() 결과를 다시 대입하지 않으면 원본이 바뀌지 않을 수 있습니다.
  • 상위 10%는 보통 quantile(0.9)입니다. 상위 10개와 다릅니다.
  • 이상/이하 조건은 경계값 포함 여부 때문에 >=, <=를 꼼꼼히 봐야 합니다.

4. 초보자가 자주 틀리는 에러 읽기

  • FileNotFoundError: CSV 경로가 틀렸습니다. 이 앱에서는 보통 data/titanic.csv입니다.
  • KeyError: 컬럼명을 잘못 썼습니다. print(df.columns)로 실제 이름을 확인합니다.
  • TypeError: 숫자 계산 대상에 문자열이 섞였거나 함수 사용법이 틀렸을 수 있습니다.
  • IndentationError: 들여쓰기가 어긋났습니다. 시험에서는 들여쓰기가 필요한 for, if를 최소화하는 것도 전략입니다.
