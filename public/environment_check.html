환경/패키지 점검

실기 시험과 연습에서 가장 허무한 실수는 코드가 아니라 환경 문제입니다. 공부 시작 전, 시험 전날, 패키지를 새로 설치한 뒤에는 이 페이지에서 import와 데이터 경로를 한 번에 확인하세요.

1. 무엇을 확인하나

Python 실행
웹 실행기가 서버에서 Python 코드를 정상 실행하는지 확인합니다.
핵심 패키지
pandas, numpy, scipy, sklearn, statsmodels, xgboost import와 버전을 확인합니다.
데이터 경로
data/titanic.csv, data/tips.csv, data/customer_churn.csv가 읽히는지 확인합니다.
간단 모델 학습
sklearn 모델이 실제로 fit/predict까지 되는지 확인합니다.

2. 실패했을 때 먼저 볼 것

  • ModuleNotFoundError: 패키지가 설치되지 않았습니다. pip install -r requirements.txt를 확인합니다.
  • FileNotFoundError: 데이터 경로가 틀렸습니다. 이 앱에서는 data/파일명.csv 형식을 사용합니다.
  • ImportError: 패키지 버전 충돌일 수 있습니다. 버전 출력 결과를 확인합니다.
  • 시간 초과: 무한 루프나 너무 무거운 모델 학습을 의심합니다.

3. 시험 전 체크 루틴

시험장에서는 긴 코드를 처음부터 쓰지 말고, 먼저 import pandas as pd, pd.read_csv(), print(df.head())가 되는지 확인하는 습관이 안전합니다.
  • 패키지 import가 되는지 확인합니다.
  • 데이터 파일 경로와 컬럼명을 확인합니다.
  • 작업형 2라면 predict_proba와 to_csv까지 짧게 테스트합니다.
  • 출력 형식이 숫자인지 CSV인지 문제 지시문을 다시 확인합니다.
