Week 6-2: 고급 데이터 전처리 (ADP 대비)

데이터 사이언티스트를 위한 궁극의 차원 관리 기법

🧭 0. 변수 관계를 왜 관리해야 할까?

ADP에서는 모델 성능뿐 아니라 변수 간 관계와 해석 가능성을 설명해야 합니다. 변수가 너무 비슷하거나 너무 많으면 모델은 학습은 할 수 있어도, 해석과 일반화에서 문제가 생길 수 있습니다.

상황 문제 대표 대응
설명변수끼리 강하게 상관됨 회귀계수 불안정, 해석 왜곡 VIF 확인, 변수 제거, 규제 회귀, 트리 모델
변수가 너무 많음 차원의 저주, 과적합, 계산 비용 증가 PCA, 변수 선택, 도메인 기반 변수 축소
변수 단위가 크게 다름 PCA/거리기반 모델에서 큰 단위 변수가 지배 StandardScaler 등 표준화
시험 감각: VIF는 회귀 해석 안정성, PCA는 차원 축소와 정보 압축을 묻는 경우가 많습니다. 둘 다 “왜 했는지”와 “무엇을 잃는지”를 써야 합니다.

👯 1. 다중공선성 (VIF)

"재판에 목격자가 2명 왔는데 둘이 똑같은 대본을 읽는 쌍둥이였습니다. 판사(모델)가 헷갈리지 않게 한 명을 집에 돌려보내는 작업입니다."

변수(X)들끼리 상관관계가 너무 높으면 회귀계수가 왜곡됩니다. VIF(Variance Inflation Factor) 값이 10을 넘으면 문제가 있다고 판단하고, 해당 변수를 제거하거나 릿지/라쏘 같은 규제 모델, 혹은 트리기반 모델(Random Forest)을 사용해야 합니다.

회귀 해석다중공선성VIF 10 기준계수 불안정

VIF는 한 설명변수가 다른 설명변수들로 얼마나 잘 설명되는지를 보는 지표입니다. VIF가 높으면 해당 변수의 정보가 다른 변수들과 많이 겹친다는 뜻이고, 회귀계수의 방향과 크기가 불안정해질 수 있습니다.

import pandas as pd
from statsmodels.stats.outliers_influence import variance_inflation_factor

X = df[['x1', 'x2', 'x3']]
vif = pd.Series(
    [variance_inflation_factor(X.values, i) for i in range(X.shape[1])],
    index=X.columns
)
print(vif)
VIF가 높다고 무조건 삭제하는 것은 위험합니다. 업무적으로 중요한 변수인지, 파생변수와 원변수가 함께 들어간 것은 아닌지, 예측 성능과 해석 목적 중 무엇이 중요한지 먼저 판단해야 합니다.

🗜️ 2. 차원 축소: 주성분 분석 (PCA)

"3D 물체를 그림자로 비추면 2D가 됩니다. 특징(변수)이 100개나 되어서 모델이 힘들어할 때, 데이터의 정보(분산)를 최대한 보존하면서 변수를 2~3개의 '주성분(Principal Component)'으로 압축하는 마법입니다."

PCA를 수행하기 전에는 반드시 StandardScaler를 이용한 데이터 정규화가 필수입니다!

차원 축소설명분산비율스케일링 필수해석력 손실

PCA는 서로 상관된 여러 변수를 새로운 축인 주성분으로 변환합니다. 첫 번째 주성분은 데이터 분산을 가장 많이 설명하고, 두 번째 주성분은 첫 번째와 직교하면서 남은 분산을 최대한 설명합니다.

from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# 1. 정규화 필수
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 2. 100개의 변수를 2개로 압축 (n_components=2)
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)
해석 포인트: explained_variance_ratio_의 합이 높을수록 축소 후에도 원래 정보가 많이 보존됩니다. 다만 주성분은 원래 변수들의 조합이라 “변수 하나의 의미”로 직접 해석하기 어렵습니다.

🧾 3. ADP 답안 작성 공식

본 분석에서는 [회귀 해석/차원 축소] 과정에서 [다중공선성/고차원 문제]를 확인하기 위해 [VIF/PCA]를 사용하였다.
VIF는 설명변수 간 정보 중복을 확인하는 지표이며, 일반적으로 10 이상이면 다중공선성을 의심한다.
PCA는 변수들을 서로 직교하는 주성분으로 변환하여 차원을 줄이고, 설명분산비율로 정보 보존 정도를 확인한다.
분석 결과 [수치]가 나타났으므로 [변수 제거/규제 모델/주성분 개수 선택]을 고려할 수 있다.
다만 변수 제거는 도메인 의미를 훼손할 수 있고, PCA는 해석력이 낮아지는 한계가 있다.

🚀 드릴 연습문제 (VIF & PCA)

[문제 1: VIF 계산] tips.csv에서 tip_rate = tip / total_bill * 100 파생변수를 만든 뒤 total_bill, tip, size, tip_rate의 VIF를 계산하세요. total_bill의 VIF를 소수점 4자리까지 출력하세요.

import pandas as pd
from statsmodels.stats.outliers_influence import variance_inflation_factor

tips = pd.read_csv('data/tips.csv')
tips['tip_rate'] = tips['tip'] / tips['total_bill'] * 100
X = tips[['total_bill', 'tip', 'size', 'tip_rate']]

vif = pd.Series(
    [variance_inflation_factor(X.values, i) for i in range(X.shape[1])],
    index=X.columns
)

print('total_bill:', round(vif['total_bill'], 4))

[문제 2: PCA 설명분산] iris.csv에서 species를 제외한 4개 변수를 표준화한 뒤 PCA(n_components=2)를 수행하세요. 두 주성분의 설명분산비율 합을 소수점 4자리까지 출력하세요.

import pandas as pd
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

iris = pd.read_csv('data/iris.csv')
X = iris.drop('species', axis=1)
X_scaled = StandardScaler().fit_transform(X)

pca = PCA(n_components=2)
pca.fit(X_scaled)

print('explained_sum:', round(pca.explained_variance_ratio_.sum(), 4))

[문제 3: ADP 서술형] PCA 2개 주성분의 설명분산비율 합이 0.9581이라고 가정합니다. PCA 사용 이유, 결과 해석, 한계를 5문장 이내로 작성하세요.

변수 간 상관성과 차원 증가로 인한 복잡도를 줄이기 위해 PCA를 적용하였다.
PCA는 원 변수를 서로 직교하는 주성분으로 변환하여 정보 손실을 최소화하면서 차원을 축소하는 방법이다.
2개 주성분의 설명분산비율 합이 0.9581이므로 원 데이터 분산의 약 95.81%가 보존되어 정보 손실이 비교적 작다고 볼 수 있다.
다만 주성분은 원 변수들의 선형 결합이므로 개별 변수의 의미를 직접 해석하기 어렵다.
따라서 예측 성능과 해석 가능성의 균형을 고려해 사용해야 한다.
