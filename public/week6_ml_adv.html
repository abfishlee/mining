머신러닝 심화 (ADP 대비)

이제 고급 분석가의 시야를 가져봅시다! 다중공선성과 클러스터링 🧐

👯 1. 쌍둥이 변수 찾아내기: 다중공선성 (VIF)

"재판에 목격자가 2명 왔는데, 알고 보니 둘이 똑같은 대본을 읽는 쌍둥이였습니다. 이럴 경우 판사(모델)는 누구의 말을 믿어야 할지 헷갈려서 오히려 판결 능력이 떨어집니다. 이 쌍둥이를 찾아내어 한 명을 집에 보내는 기술이 VIF(분산팽창요인)입니다."

회귀 분석을 할 때 입력 변수(X)들끼리 서로 너무 강하게 얽혀 있으면 모델이 망가집니다. ADP 서술형에 반드시 등장하는 개념입니다.

from statsmodels.stats.outliers_influence import variance_inflation_factor
import pandas as pd

# 팁 데이터를 활용해 봅시다.
df = pd.read_csv('data/tips.csv')

# 숫자형 변수만 추출 (total_bill, tip, size)
X = df[['total_bill', 'tip', 'size']]

# VIF 계산 함수 (값이 10을 넘어가면 심각한 다중공선성이 있다고 판단!)
vif = pd.DataFrame()
vif["VIF Factor"] = [variance_inflation_factor(X.values, i) for i in range(X.shape[1])]
vif["features"] = X.columns
print(vif)

💼 현업 선배의 실무 노하우

🚨 파생변수를 만들 때 다중공선성 주의!

많은 초보자들이 총액 = 물건값 + 세금 이라는 파생변수를 만든 뒤, 모델에 [총액, 물건값, 세금] 세 개의 열을 모두 넣어버립니다. 이는 완벽한 다중공선성을 유발하여 모델의 예측력을 박살냅니다.

  • 실무 정석: 파생변수(총액)를 만들었으면, 원래 변수(물건값, 세금)는 과감히 버리거나(Drop), VIF를 돌려서 10 이상인 녀석을 골라내야 합니다.
  • 원핫 인코딩 시: pd.get_dummies(drop_first=True) 옵션을 주어 열 하나를 삭제하는 것도 다중공선성을 막기 위한 필수 테크닉입니다.

끼리끼리 모여라: K-Means 군집화 (비지도학습)

"운동장에 모인 100명의 학생들을 보고, 체육 선생님이 '비슷한 덩치끼리 3개 조로 모여!'라고 명령합니다. 정답(Label)이 없어도 컴퓨터가 알아서 데이터들을 K개의 그룹으로 묶어주는 마법입니다." 
from sklearn.cluster import KMeans
import pandas as pd

# 붓꽃 데이터를 불러옵니다. 이번엔 정답(종)을 모른다고 가정해봅시다!
iris = pd.read_csv('data/iris.csv')
X = iris[['sepal_length', 'sepal_width', 'petal_length', 'petal_width']]

# 3개의 덩어리(클러스터)로 묶어달라고 모델에게 명령합니다.
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
iris['cluster'] = kmeans.fit_predict(X)

print(iris[['species', 'cluster']].head(10))
# 기계가 분류한 'cluster' 숫자와 실제 'species'가 얼마나 잘 매칭되는지 확인!

🚀 드릴 연습문제 (머신러닝 심화)

[문제 1] K-Means 군집화에서 K(클러스터 개수)를 몇 개로 정할지 모를 때 사용하는 방법이 Elbow Method(팔꿈치 기법)입니다. 클러스터 개수를 1부터 5까지 늘려가며 model.inertia_ (오차의 합)를 구하여 리스트로 출력해 보세요. (값이 확 꺾이는 부분이 바로 적절한 K입니다!)

import pandas as pd
from sklearn.cluster import KMeans

iris = pd.read_csv('data/iris.csv')
X = iris.drop('species', axis=1)

inertias = []
for k in range(1, 6):
    model = KMeans(n_clusters=k, random_state=42, n_init=10)
    model.fit(X)
    inertias.append(round(model.inertia_, 2))

print("K=1~5 일 때의 오차합(Inertia):", inertias)
# 결과가 [681.37, 152.35, 78.85, 57.23, 46.45] 처럼 나옵니다.
# 2에서 3으로 갈 때 팍 꺾이고, 3 이후부터는 완만해집니다. 즉 적정 K는 3! (팔꿈치)
